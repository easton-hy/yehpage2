PS Web - Gallery

Today, I used a Javascript algrithm to achieve auto resize photo's size to let all of the photo will be resized to the same height. The origin template can fill the photo to the same weight to arrange all the photos to 4 columns, but the height is different, which causes the photos are messy. Even thought the algrithm can resize the photo once the photos are loaded, the page still is arranged by the photos' size before resizing. The outcome is that all of photos are resized to the same height, but the photos are still arranges messy.

To figure out the problem, the strategy is to create the blank photos with the same size and place on the page, this will avoid the photos are arranged messy. Then, the images will be loaded and be resized to fit well with the blank photos. The Javascript is shown below:

Javascript for resizing photo

<script>
	document.addEventListener("DOMContentLoaded", function() {
		var placeholders = document.querySelectorAll('.image-placeholder');

		placeholders.forEach(function(placeholder) {
			// 假设占位符宽度与其父元素相同
			var width = placeholder.parentElement.clientWidth;
			var height = width * 9 / 16;  // 根据16:9的比例计算高度

			// 设置占位符的初始样式
			placeholder.style.width = width + 'px';
			placeholder.style.height = height + 'px';
			placeholder.style.backgroundColor = 'transparent'; // 可以设置为任意占位色

			var imgSrc = placeholder.getAttribute('data-src');
			var img = new Image();

			img.onload = function() {
				placeholder.innerHTML = ''; // 清空占位符
				placeholder.appendChild(this); // 添加图片
				adjustImage(this);  // 调整图片大小
			};

			img.src = imgSrc;
		});
	});

	function adjustImage(img) {
		var gridItem = img.parentElement;
		var containerWidth = gridItem.clientWidth;
		var containerHeight = containerWidth * 9 / 16; // 假定16:9的比例
		var imageAspectRatio = img.naturalWidth / img.naturalHeight;

		img.style.position = 'absolute';
		img.style.top = '50%';
		img.style.left = '50%';
		img.style.transform = 'translate(-50%, -50%)';

		if (imageAspectRatio < 16 / 9) {
			img.style.height = containerHeight + "px";
			img.style.width = 'auto';
		} else {
			img.style.width = containerWidth + "px";
			img.style.height = 'auto';
		}
	}
</script>

PS Web - Password Page

The problems are:

  1. The developer mode will shows the determine statement which indicates the correct password.
  2. The statement will also indicate the url.
  3. My page does have the server to storage different password for each users.
To address this problem, I encrypted the password and the following url. First, I used the user name and '.' as the determine statement, meaning the users are required to input a user name and the input user name will a the first part of the corresponding password, e.g. the user name is 'yeh' and the password's first part should be 'yeh.', if not, the determine statement will show the 'Password is incorrect...'. The latter part of the password is the name of the following url's name. By this way, the correct password is depend on the name of the html page and the user name. If the latter part of input password isn't correct name of page, the page jump to the page with the latter part of the password and the page isn't exist so that the password is hard to decipher.

Javascript for password

<script>
function redirectToPage() {
	var username = document.getElementById("account").value;
	var password = document.getElementById("passwordInput").value;
	var redirectBase = "../blogpage/";
	var houzhuoBase = ".html";

	var correctPasswordFormat = username + ".";

	if (password.startsWith(correctPasswordFormat)) {
		var remainingPart = password.substring(username.length + 1);
		var fullPath = redirectBase + remainingPart+houzhuoBase;

		console.log("Redirecting to: " + fullPath); 

		window.location.href = fullPath;
	} else {
		alert("Password is incorrect..."); 
	}
	return false;
}
</script>
